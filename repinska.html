JavaScript Variables

Variables are containers for storing data values


                            // Variable declarations
                            let message = "Hello, World!";
                            const PI = 3.14159;
                            var oldVariable = "Deprecated way";

                            // Using variables
                            console.log(message); // Output: Hello, World!
                            console.log(PI);      // Output: 3.14159

Variable Types

let

Block-scoped, can be reassigned


                                let count = 10;
                                count = 20; // OK

const

Block-scoped, cannot be reassigned


                                const gravity = 9.8;
                                gravity = 10; // Error!

JavaScript Operators

Operators are used to perform operations on variables and values

Types of Operators

Operator Type Examples
Arithmetic + - * / % **
Assignment = += -= *= /=
Comparison == === != !== > <
Logical && || !

Operator Examples


                            // Arithmetic operators
                            let sum = 10 + 5;        // 15
                            let product = 10 * 5;    // 50
                            let remainder = 10 % 3;  // 1

                            // Comparison operators
                            let isEqual = (10 == "10");   // true (loose equality)
                            let isStrictEqual = (10 === "10"); // false (strict equality)

                            // Logical operators
                            let isAdult = true;
                            let hasLicense = false;
                            let canDrive = isAdult && hasLicense; // false

JavaScript Expressions

An expression is a combination of values, variables, and operators


                        // Different types of expressions
                        let arithmetic = 5 * (10 + 3); // 65
                        let stringExp = "Hello" + " " + "World"; //"Hello World"
                        let booleanExp = (5 > 3);   // true
                        let functionCall = Math.max(10, 20, 15); // 20

JavaScript Keywords

Reserved words that have special meaning in JavaScript

Declaration Keywords
  • let - declare variables
  • const - declare constants
  • var - old way to declare variables
  • function - declare functions
  • class - declare classes
Control Flow Keywords
  • if, else
  • for, while, do
  • switch, case
  • break, continue
  • return

JavaScript Comments

Comments are used to explain code and make it more readable


                            // This is a single-line comment
                            /*
                            This is a multi-line comment
                            It can span multiple lines
                            */
                            let price = 100; // Price in dollars
                            /**
                            * Function to calculate total price
                            * @param {number} quantity - Number of items
                            * @returns {number} Total price
                            */
                            function calculateTotal(quantity) {
                                return price * quantity;
                            }

Commenting Best Practices

  • Explain why, not what
  • Use JSDoc for functions
  • Keep comments up-to-date
  • Avoid obvious comments
  • Use comments for complex logic

JavaScript Identifiers

Names used to identify variables, functions, objects, etc.

Valid Identifiers


                            let firstName;
                            let _privateVar;
                            let $price;
                            let user123;
                            let camelCaseName;

Invalid Identifiers


                            let 123user;      // Cannot start with number
                            let first-name;   // Hyphen not allowed
                            let let;          // Reserved keyword
                            let first name;   // Space not allowed

Case Sensitivity

JavaScript is CASE-SENSITIVE!


                        let myVariable = "Hello";
                        let myvariable = "World";

                        console.log(myVariable); // "Hello"
                        console.log(myvariable); // "World"
                        // These are TWO DIFFERENT variables!

                        // Function names are also case-sensitive
                        function calculateTotal() { /* ... */ }
                        function CalculateTotal() { /* ... */ }
                        // These are TWO DIFFERENT functions!

Camel Case Convention

The standard naming convention in JavaScript


                            // Variables and functions - camelCase
                            let firstName = "John";
                            let totalAmount = 100;
                            function calculateTotalPrice() { /* ... */ }

                            // Constants - UPPER_SNAKE_CASE
                            const MAX_USERS = 100;
                            const API_KEY = "abc123";

                            // Classes - PascalCase
                            class UserAccount { /* ... */ }
                            class ShoppingCart { /* ... */ }

                            // Private variables - _leadingUnderscore
                            let _privateData = "secret";

Naming Conventions Comparison

Type Convention Example
Variables camelCase userName
Functions camelCase getUserData()
Classes PascalCase UserProfile
Constants UPPER_SNAKE_CASE MAX_SIZE
