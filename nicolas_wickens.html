css

in the following sections, you will see what I learn about the css.

- CSS -

CSS = Cascading Style Sheets

CSS is used to style HTML elements

CSS describes how HTML elements are to be displayed on screen

here are some tips when I use CSS:

  • hover is very good for UX
  • id will be the last to use
  • name the id well
  • flex box are powerful

- hover is very good for UX -

when you want to make your website more interactive, use hover effect.

it will give user feedback when they interact with elements.

for example: change color, size, or add animation on hover.

back

- id will be the last to use -

when you want to style an element, try to avoid using id selector first.

because id has the highest specificity.

so if you use id first, it will be hard to override it later.

back

- name the id well -

when you use id selector, make sure to name it well.

because id should be unique in the page.

so use meaningful names that describe the element.

back

- flex box are powerful -

flex box is a powerful layout module in CSS.

it makes it easier to design flexible responsive layout structure.

so learn flex box well to make your layout better.

- JavaScript -

JavaScript is a programming language.

JavaScript is used to make web pages interactive.

JavaScript can update and change both HTML and CSS.

here are some tips when I use JavaScript:

  • write debug messages
  • write comments
  • understand asynchronous programming

- write debug messages -

when you coding, sometime you will encounter bugs or errors.

so write debug messages to help you find the problem easily.

use console.log() to print messages to the console.

back

- write comments -

write comments in your code to explain what it does.

this will help you and others understand the code better.

use // for single-line comments and /* */ for multi-line comments.

back

- understand asynchronous programming -

JavaScript is single-threaded, but it can handle asynchronous tasks.

learn about callbacks, promises, and async/await to manage async code.

this will help you write more efficient and responsive applications.
